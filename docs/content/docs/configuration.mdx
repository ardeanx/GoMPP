GoMPP uses a layered configuration system:

1. **Environment variables** (prefix `GOMPP_`) — Set at startup, requires restart to change
2. **`.env` file** — Automatically loaded from the project root
3. **System Settings** (database) — Configurable at runtime via the admin dashboard

Environment variables take precedence over `.env` file values.

---

## Environment Variables

### JWT (Required)

| Variable                      | Default | Description                                                                                 |
| ----------------------------- | ------- | ------------------------------------------------------------------------------------------- |
| `GOMPP_JWT_SECRET`            | —       | **Required.** HMAC-SHA256 signing key for JWT tokens. Generate with `openssl rand -hex 32`. |
| `GOMPP_JWT_ACCESSEXPIRY_MIN`  | `60`    | Access token lifetime in minutes                                                            |
| `GOMPP_JWT_REFRESHEXPIRY_DAY` | `7`     | Refresh token lifetime in days                                                              |

<Callout type="error">
  The server will refuse to start if `GOMPP_JWT_SECRET` is not set.
</Callout>

## System Settings (Database)

| Key            | Type   | Default         | Description                                     |
| -------------- | ------ | --------------- | ----------------------------------------------- |
| `site_name`    | string | `"GoMPP"`       | Application display name                        |
| `site_logo`    | string | `"/gompp.webp"` | Logo URL (used in sidebar, auth pages, landing) |
| `site_favicon` | string | `"/gompp.webp"` | Favicon URL                                     |
| `theme_color`  | string | —               | Primary theme color as hex (e.g., `#6366f1`)    |
| `base_url`     | string | —               | Public-facing base URL                          |

### Player Tab

| Key                        | Type    | Default       | Description                          |
| -------------------------- | ------- | ------------- | ------------------------------------ |

| `player_autoplay`          | boolean | `false`       | Auto-play videos                     |
| `player_muted`             | boolean | `false`       | Start muted                          |
| `player_loop`              | boolean | `false`       | Loop playback                        |
| `player_controls`          | boolean | `true`        | Show player controls                 |
| `player_volume`            | number  | `1.0`         | Default volume (0.0 – 1.0)           |
| `player_color`             | string  | —             | Player accent color                  |
| `player_logo`              | boolean | `false`       | Show logo overlay                    |
| `player_logo_url`          | string  | —             | Logo image URL                       |
| `player_logo_position`     | string  | `"top-right"` | Logo position                        |
| `player_watermark`         | boolean | `false`       | Show watermark                       |
| `player_poster_url`        | string  | —             | Default poster image                 |
| `player_subtitles_enabled` | boolean | `true`        | Enable subtitle support              |

## Configuration Loading Order

GoMPP uses [Viper](https://github.com/spf13/viper) with this resolution order (highest priority first):

1. **Environment variables** — Prefixed with `GOMPP_`, underscores replace dots (e.g., `GOMPP_DATABASE_HOST`)
2. **`.env` file** — Loaded via `gotenv.OverLoad()` before Viper initializes
3. **Config file** — Optional `config.yaml` in the working directory or `./config/`
4. **Defaults** — Hardcoded fallbacks in the config package

For system settings (database-backed), changes take effect:

- **Immediately** for settings read on each request (branding, player config)
- **Within 30 seconds** for rate limits (polled by background goroutine)
- **On restart** for worker count (read once at startup, unless overridden by `max_concurrent_jobs`)
